- Using "flask run" we can run the app locally.
- A simple Dockerfile, use below commands to run the app:
docker build -t flask-app1 .
docker run -p 5001:5001 flask-app1

(Did docker compose and env configurations before this simple dockerfile and then realized its not needed in my case and hence deleted everything. Running on port 5001 instead of 5000, because my port 5000 is busy).

- After preparing the terraform files, run commands:
terraform init
terraform plan
terraform apply
- Thought alot about what kind of terraform structure to use, I used different modules for each resource (each having a main.tf, variables.tf, and outputs.tf). Then I have an environment folder that contains the dev and prod environments. This is where I actually define the variables in the main.tf file (didn't use .tfvars file because its not really needed for the simple app I have). 

- The infrastructure was successfully provisioned.
- Then used the command:
az aks get-credentials --resource-group rgdev1 --name aks-flaskdev1
kubectl get nodes

- This returned the 2 nodes. Showing the infrastructure was provisioned properly.

- switch to the app directory and use command:
az acr login --name crflaskdev1
- This resulted in ssl connectivity errors, due to the Zscaler proxy. Hence took the .pem file of Zscaler Root CA and appended it to the end of the cacert.pem file in C:\Program Files\Microsoft SDKs\Azure\CLI2\Lib\site-packages\certifi
- Then the login succeeded. Did 3 hours of research to figure this one thing out. Phewww

- Next building the docker image, tagging and pushing:
docker build -t flask-api:dev .
docker tag flask-api:dev crflaskdev1.azurecr.io/flask-api:dev
docker push flaskacrdev123.azurecr.io/flask-api:dev

- Now the image is pushed to ACR

- Next, created the deployment and service files, and ran the following commands to deploy and get the IP that the service is running on:
kubectl apply -f k8s/
kubectl get svc flask-app-service

- Take the IP that the command provides and access the flask app.
